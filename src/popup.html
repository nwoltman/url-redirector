Notes:

The "Requested URL" input is a Regular Expression (JavaScript style), but regex syntax is usually not necessary since regular URLs (i.e. "www.google.com") will work exactly as expected.

"^https?://" is automatically prepended to the "Requested URL" input when creating the regex, so it should not be in your input URL.

Query parameters (everything from "?" and on in the URL) will be kept when a redirection occurs.
